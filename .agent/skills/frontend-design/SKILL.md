# Frontend Design Mandate

This skill guides the creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. 

## When to Trigger
This skill MUST be consulted when:
- Creating NEW frontend components or pages.
- Modifying EXISTING styles (CSS, Tailwind).
- Adding ANIMATIONS (Framer Motion, CSS).
- Discussing UX/UI improvements.

## Operational Mandates

1. **Commit to a BOLD Aesthetic**: Choose a conceptual direction (e.g., brutally minimal, editorial, maximalist) and execute it with precision. Avoid "default" look.
2. **Typography is UI**: NEVER use system fonts or Inter by default unless requested. Pair a distinctive display font with a functional body font.
3. **Motion is Experience**: Proactively add micro-interactions and entrance animations (e.g., staggered reveals) that create delight.
4. **Layout as Differentiation**: Break the grid. Use asymmetry, overlaps, and generous/intentional negative space.

## Negative Constraints (What to NEVER do)
- **NO Purple Gradients on White**: Avoid the "Generic Tech Startup" color palette.
- **NO Default Components**: Don't just spit out a standard Shadcn/UI component without customizing its character to match the project's vibe.
- **NO "Standard" Spacing**: Avoid predictable `p-4` spacing; use varying intensities and intentional gaps to create rhythm.

## Performance Standards
A successful implementation should:
- Look "hand-designed" rather than "AI-generated".
- Feel responsive and alive with interaction.
- Use a cohesive, intentional color system.
- Match implementation complexity to the aesthetic vision.
